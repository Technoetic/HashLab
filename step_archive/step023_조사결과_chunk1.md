# Step 023 디자인 패턴 분석 chunk1

## Step-Back

- 목적: 해시 함수 튜토리얼에 적용할 디자인 패턴을 결정 가능 수준으로 정리
- 영향: step025 기획, step030 설계
- 핵심 항목: 레이아웃 / 색상·타이포 / 인터랙션

## 조사 축 1: 레이아웃

### 후보 A — 좌측 네비, 우측 콘텐츠 (Visualgo 패턴)

- 좌측 240px 사이드바: 챕터 목록
- 우측: 챕터 콘텐츠(설명 + 인터랙티브 시각화)
- 장점: 챕터 이동 즉시 가능, 진행 상태 표시
- 단점: 모바일에서 햄버거 메뉴 필요

### 후보 B — 단일 컬럼 스크롤 (긴 글 친화)

- 모든 챕터를 위에서 아래로 스크롤
- TOC가 우측에 sticky
- 장점: 모바일 친화, 자연스러운 학습 흐름
- 단점: 긴 페이지 부담

### 후보 C — 풀스크린 챕터 + 화살표 네비

- 한 번에 한 챕터만 보이고 → 진행
- 장점: 한 개념에 집중
- 단점: 비교가 어려움

→ 비교 자료만 제시. 결정은 step025/step030에서.

## 조사 축 2: 색상

### 후보 A — Dark OLED + 단일 강조

- 60% `#0a0a0a`, 30% `#1a1a1a` 카드 + `#e6e6e6` 텍스트, 10% `#00ff95`
- 코드/해시 출력 시인성 매우 좋음
- AI Slop 회피: 보라 그라데이션 X, 무지개 X

### 후보 B — Solarized Light

- 60% `#fdf6e3`, 30% `#eee8d5` + `#073642` 텍스트, 10% `#268bd2`
- 장시간 학습에 부담 적음
- 단점: 강조색 대비 약함

→ 본 프로젝트는 후보 A 권장(코드/해시 시각화 친화).

## 조사 축 3: 타이포그래피

- UI 본문: Helvetica Neue / system-ui
- 코드/해시: JetBrains Mono / Courier New
- 제목: Helvetica Neue 700, 본문 400
- 폰트 크기 최대 4단계: 12 / 14 / 16 / 24 / 40 (modular scale)
- 폰트 두께 최대 2: 400, 700

## AI Slop 방지

- ❌ Inter 사용 X (트렌디 클리셰)
- ❌ 보라 그라데이션 X
- ❌ 무지개 hue rotate X
- ❌ 과도한 border-radius (8px 이상 카드 X)
- ✅ 직각 + 1px 라인 강조 (Brutalism)
