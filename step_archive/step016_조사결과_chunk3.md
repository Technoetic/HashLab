# Step 016 조사결과 chunk3 - 인터랙티브 시각화 패턴

## 1. 입력 슬롯 + 실시간 다이제스트

- `<input>` 텍스트 박스 → 키 입력마다 SHA-256/MD5 즉시 계산
- 출력은 hex로 표시, 각 hex 글자에 색상 매핑(시각적 패턴 인식)
- Web Crypto API: `crypto.subtle.digest('SHA-256', bytes)` 로 브라우저 내장 사용

## 2. Avalanche 비교기

- 두 입력 박스(왼쪽/오른쪽) → 1비트 차이 입력
- 두 해시 출력의 비트별 일치/불일치를 색상으로 표시
- "한 글자만 바꿔도 50% 비트가 뒤집힌다" 시각화

## 3. 충돌 시뮬레이터(약식 해시)

- 일부러 작은 출력(예: 8 bit) 해시로 줄임 → 적은 입력으로도 충돌 발생
- 사용자가 입력을 추가할 때마다 어느 버킷에 들어가는지 애니메이션
- 충돌 발생 순간 빨간 효과

## 4. 해시 테이블 박스 시각화

- 16개 버킷(0~15) 그리드 → 사용자가 단어 입력 시 해시(mod 16) → 해당 버킷에 카드 떨어짐
- 충돌 시 chaining(체인) 또는 open addressing(이웃 칸으로) 시각적 데모
- O(1) 조회의 직관적 이해

## 5. 비밀번호 저장 데모

- 사용자가 비밀번호 입력 → "DB에 저장될 값" 출력
- bcrypt 파라미터(saltRounds) 슬라이더 → 연산 시간 변화 그래프
- "왜 평문이 위험한가" 옆 박스에 평문 vs 해시 비교

## 6. Git 커밋 시뮬레이터

- 텍스트 박스에 코드 → 1바이트 수정 시 커밋 해시가 어떻게 바뀌는지 비교
- 두 커밋 해시 옆에 diff 표시

## 7. 비트코인 마이닝 미니게임

- 입력에 nonce 자동 증가 → SHA-256(SHA-256(...)) 반복
- 결과 해시 앞 0이 N개 이상이면 "발견!" 효과
- 난이도 슬라이더로 N 조정 → 평균 시도 수 차트

## 8. Bloom Filter 인터랙티브

- 32 bit 벡터 시각화 → "추가" 시 k개 위치 점등, "조회" 시 모두 점등돼 있으면 PROBABLY YES
- False positive 발생 케이스 보여주기

## 9. 권장 인터랙션 원칙

- 즉시 반응(Live Reactive): 입력→출력 100ms 내 갱신
- 색상 코드: 일치/불일치 명확 대비
- 단계적 노출: 챕터 잠금 해제 방식, 한 번에 한 개념
- 모바일 터치 44×44pt 이상
