# Step 002 Context 전략 - Chunk 2: 서브에이전트/파일/Context 예산

## 서브에이전트 전략

- 동시 실행 상한: 10개
- 권장 동시: 5~8개 (SMALL 프로젝트)
- 모델: haiku (일반 조사) / sonnet (평가/디자인 검증)
- 의존성 순서: 조사 → (대기) → 종합 → 구현 (병렬 금지 시점은 조사+구현 동시)

## 파일 읽기 전략

- 500줄 이상 파일 전체 Read 금지
- Grep으로 키워드/심볼 위치 파악 → offset+limit으로 부분 Read
- 같은 파일 재읽기 금지 (메모리 의존)
- 청크 파일은 500줄 이하 유지 (research-validator.ps1 통과 조건)

## Context 예산

- 메인 에이전트: 동시 활성 파일 10개 이하
- 서브에이전트: 1~5개 파일 이하
- Bash output: head/tail/grep로 사전 축약
- 90% 도달 시 잔여 작업 서브에이전트 위임

## 분기 결정 트리

```
파일 수 < 100   → 5~10 서브에이전트
파일 수 100~1000 → 10~20 서브에이전트
파일 수 > 1000   → 20+ 서브에이전트
```

현재 프로젝트(82): **5~10 서브에이전트** 적용.
