  3. 인터랙티브 시각화 패턴(슬라이더/충돌 시뮬레이터/버킷 테이블)

## 1. 해시 함수란?

해시 함수 H는 임의 길이 입력 x를 받아 **고정 길이 출력 H(x)** 를 만드는 함수다.

### 4대 성질

| 성질 | 설명 | 직관적 비유 |
|:---|:---|:---|
| **결정성(Deterministic)** | 같은 입력 → 항상 같은 출력 | "같은 주민번호엔 항상 같은 사람" |
| **균등성(Uniform)** | 출력이 가능한 범위에 골고루 퍼짐 | "공평한 주사위" |
| **단방향성(One-way)** | 출력에서 입력을 역산할 수 없음 | "갈은 고기로 소를 복원 불가" |
| **충돌 저항성(Collision Resistance)** | 다른 입력이 같은 출력 만들기 어려움 | "다른 사람이 같은 지문 가질 확률 ≈ 0" |

### Avalanche 효과(눈사태)

입력 1비트만 바뀌어도 출력이 약 50% 비트가 뒤집힌다. "hello" → "hellp"만 바꿔도 SHA-256은 완전히 달라 보인다.

### 출력 길이

- MD5: 128 bit (32 hex)
- SHA-1: 160 bit (40 hex)
- SHA-256: 256 bit (64 hex)
- 입력이 1글자든 1GB이든 출력 길이는 동일.

## 2. 해시 vs 암호화 vs 인코딩 차이

| 기법 | 가역 | 키 필요 | 출력 길이 | 용도 |
|:---|:---|:---|:---|:---|
| 해시 | ✗ | ✗ | 고정 | 무결성/식별/검색 |
| 대칭 암호 | ✓ | ✓ | 가변 | 기밀성 |
| Base64 | ✓ | ✗ | 가변 | 표현 변환 |

초보자가 가장 자주 헷갈리는 지점이다. 튜토리얼에선 이 표를 1챕터에 시각화로 제시한다.
