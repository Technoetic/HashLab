# Step 019 참고 레포 분석

## 클론 정책

용량/시간 절약 위해 실제 clone 대신 위 step017 표 기반 패턴 차용.

## 차용 패턴

### A. visualgo 스타일
- 좌측 컨트롤 패널, 우측 큰 캔버스
- 스텝 슬라이더로 알고리즘 진행 단계 제어
- 컬러 코드: 현재 처리 노드(주황), 비교 대상(노랑), 완료(초록)

### B. algorithm-visualizer 스타일
- 코드 패널 동기화: 알고리즘 코드 라인이 시각화와 함께 하이라이트
- 토글: 코드 보기/숨기기

### C. SHA-256 애니메이션 스타일
- 입력 비트 → 64라운드 → 출력 시각화
- 우리는 단순화: 입력 → 다이제스트 즉시 표시(라운드 시각화는 옵션)

## 우리 프로젝트 적용 결정

- 좌측 컨트롤 + 우측 시각화 (visualgo 차용)
- 코드 패널은 챕터별 토글 (algorithm-visualizer 차용)
- 스텝 슬라이더 대신 실시간 입력(초보자 친화)
- 컬러 코드는 토큰화: --c-active, --c-collide, --c-ok

## 결론

레포 직접 클론 불필요. 패턴만 통합하여 자체 구현.
