# Step 023 디자인 패턴 분석 chunk2

## 조사 축 4: 간격(Spacing)

8px 그리드 기반:
- 4px(미세 정렬) / 8px(아이콘-텍스트) / 16px(컴포넌트 내부 패딩)
- 24px(섹션 내) / 32px(섹션 간) / 64px(챕터 간)

패딩 ≤ 마진 원칙 유지.

## 조사 축 5: 인터랙션

### 후보 A — Live Reactive (즉각 반응)

- 입력 키 다운마다 즉시 해시 재계산 (디바운스 0~50ms)
- 장점: 학습자가 인과관계를 즉각 인지
- 단점: 무거운 연산은 불가(SHA-256은 무관, OK)

### 후보 B — Step-by-Step (버튼 클릭 진행)

- "다음" 버튼으로 한 단계씩
- 장점: 알고리즘 내부를 단계별로 이해
- 단점: 클릭 피로

→ 본 프로젝트는 후보 A 기본 + 일부 챕터에서 B 옵션 제공.

## 조사 축 6: 마이크로 인터랙션

- 해시 글자 등장 애니메이션: 한 글자씩 페이드인(80ms 간격)
- 충돌 시: 카드가 흔들리며 빨강 outline (0.3s)
- Avalanche 비교: 다른 비트만 노란색 하이라이트

## 조사 축 7: 접근성

- 모든 인터랙티브 요소 키보드 포커스 가능
- focus-visible: 2px outline, 강조색
- 색상 대비 ≥ 4.5:1 (Dark OLED 강조색 #00ff95 vs #0a0a0a = 14.5:1, OK)
- 터치 타겟 ≥ 44×44pt

## 조사 축 8: 반응형

표준 브레이크포인트:
- mobile: ~640px
- tablet: 641~1024px
- desktop: 1025px+

레이아웃 변환:
- desktop: 좌 사이드바 + 우 콘텐츠
- tablet/mobile: 상단 햄버거 메뉴 + 단일 컬럼
